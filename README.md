# 📚 Information Aggregator & Search Platform

ระบบค้นหาและรวมรวมข้อมูลอัจฉริยะจากแหล่งข้อมูลหลายแหล่ง

## ✨ ภาพรวม (Overview)

Information Aggregator & Search Platform เป็นเว็บแอปพลิเคชันที่ออกแบบมาเพื่อช่วยผู้ใช้ในการค้นหา วิเคราะห์ และรวมรวมข้อมูลจากแหล่งต่างๆ ในลักษณะที่ทันสมัย มีประสิทธิภาพ และใช้งานง่าย

---

## 🎯 ฟีเจอร์หลัก (Key Features)

### 1️⃣ ระบบค้นหาข้อมูลและเอกสาร
- **ค้นหาแบบสอบค้น (Advanced Search)**: ค้นหาด้วยคำสำคัญ ตัวกรอง และเงื่อนไขที่ซับซ้อน
- **ค้นหาแบบอัจฉริยะ (Smart Search)**: ใช้ AI/ML เพื่อให้ผลลัพธ์ที่เกี่ยวข้องมากขึ้น
- **การกรองข้อมูล (Filtering)**: กรองตามวันที่ ประเภท แหล่งที่มา และคณลักษณะอื่นๆ
- **ค้นหาแบบเรียลไทม์ (Real-time Search)**: ผลลัพธ์ปรากฏทันที่ขณะพิมพ์

### 2️⃣ สรุปและรวมรวมเอกสารข้อมูล
- **สรุปอัตโนมัติ (Auto Summary)**: สรุปเอกสารยาวๆ ให้เป็นประเด็นสำคัญ
- **รวมข้อมูลข้ามแหล่ง (Cross-Source Aggregation)**: นำข้อมูลจากหลายแหล่งมารวมกันอย่างสมเหตุสมผล
- **วิเคราะห์เปรียบเทียบ (Comparative Analysis)**: เปรียบเทียบข้อมูลจากแหล่งต่างๆ
- **ความเชื่อมโยง (Data Linking)**: แสดงการเชื่อมโยงและความสัมพันธ์ระหว่างข้อมูล

### 3️⃣ ส่วนติดต่อผู้ใช้ที่ทันสมัย (Modern UI/UX)
- **ดีไซน์ Responsive**: ทำงานได้ดีบนทุกอุปกรณ์ (Desktop, Tablet, Mobile)
- **ลำดับขั้นอิเลกทรนิกส์เหตุผล (Intuitive Navigation)**: ง่ายต่อการเรียนรู้และใช้งาน
- **ธีมสีสวย (Beautiful Themes)**: ธีมสว่าง/มืด และแพลตฟอร์มเหมาะสม
- **ความเร็วสูง (High Performance)**: โหลดเร็ว การตอบสนองทันที

## 🚀 การติดตั้งและการตั้งค่า (Installation & Setup)

### ข้อกำหนดเบื้องต้น (Prerequisites)
```bash
• Node.js v16+ หรือ Python 3.8+
• npm หรือ yarn
• Docker & Docker Compose (optional)
• Git
```

### วิธีการติดตั้ง (Quick Start)

#### 1. Clone Repository
```bash
git clone https://github.com/yourusername/information-aggregator.git
cd information-aggregator
```

#### 2. ติดตั้งส่วนหน้า (Frontend Setup)
```bash
cd frontend
npm install
# หรือ
yarn install

# ตั้งค่าตัวแปรสภาพแวดล้อม
cp .env.example .env.local

# เรียกใช้เซิร์ฟเวอร์การพัฒนา
npm run dev
```

#### 3. ติดตั้งส่วนหลัง (Backend Setup)
```bash
cd ../backend
npm install
# หรือ
pip install -r requirements.txt

# ตั้งค่าตัวแปรสภาพแวดล้อม
cp .env.example .env

# เรียกใช้เซิร์ฟเวอร์
npm run dev
# หรือ
python app.py
```

#### 4. ตั้งค่าฐานข้อมูล (Database Setup)

**ตัวเลือก A: ใช้ Local SQLite Database (แนะนำสำหรับการพัฒนา)**
```bash
cd ../database

# สร้างไฟล์ฐานข้อมูล local
npm run init-local-db

# เรียกใช้ migrations
npm run migrate:local

# นำเข้าข้อมูลตัวอย่าง (ตัวเลือก)
npm run seed:local
```

**ตัวเลือก B: ใช้ Local MongoDB (ต้อง MongoDB Community Edition)**
```bash
# ตรวจสอบว่า MongoDB ทำงานอยู่
mongod

# ในไดเรกทอรี่ที่แยกต่างหาก - เรียกใช้ MongoDB local
cd path/to/mongodb/bin
./mongod --dbpath path/to/local/data

# ในโปรเจค backend
npm run migrate:mongo
npm run seed:mongo
```

**ตัวเลือก C: ใช้ Local PostgreSQL**
```bash
# สร้างฐานข้อมูล local (สมมติ PostgreSQL ติดตั้งแล้ว)
createdb information_aggregator_local

# เรียกใช้ migrations
npm run migrate:postgres

# นำเข้าข้อมูลตัวอย่าง
npm run seed:postgres
```

**ตัวแปรสภาพแวดล้อม (.env) สำหรับ Local Database:**
```env
# SQLite (แนะนำ)
DB_TYPE=sqlite
DB_PATH=./data/database.sqlite

# หรือ MongoDB Local
# DB_TYPE=mongodb
# DB_CONNECTION_STRING=mongodb://localhost:27017/information_aggregator_local

# หรือ PostgreSQL Local
# DB_TYPE=postgresql
# DB_HOST=localhost
# DB_PORT=5432
# DB_NAME=information_aggregator_local
# DB_USER=postgres
# DB_PASSWORD=password
```

#### 5. เรียกใช้ด้วย Docker (Alternative - Docker)
```bash
docker-compose up -d
```

## �️ คู่มือ Local Database (Local Database Guide)

### 📌 ทำไมต้องใช้ Local Database?
- ✅ ไม่ต้องเชื่อมต่ออินเทอร์เน็ต
- ✅ ฉลาดสำหรับการพัฒนา (Development)
- ✅ ความเร็วสูง
- ✅ ไม่มีค่าใช้บริการ
- ✅ ข้อมูลยังคงอยู่บนเครื่องของคุณ

### 1. SQLite (แนะนำสำหรับผู้เริ่มต้น)

**ข้อดี:**
- ไม่ต้องติดตั้งเซิร์ฟเวอร์แยกต่างหาก
- ไฟล์เดียว สำหรับเก็บข้อมูลทั้งหมด
- เหมาะสำหรับการพัฒนาและทดสอบ

**ข้อเสีย:**
- ไม่เหมาะสำหรับข้อมูลขนาดใหญ่มาก
- ไม่สนับสนุน concurrent access เยอะๆ

**ติดตั้ง:**
```bash
# Windows/Mac/Linux - ไม่ต้องติดตั้งอะไรเพิ่มเติม
# SQLite ติดตั้งมากับ Python หรือ Node.js แล้ว

# ตรวจสอบเวอร์ชัน
sqlite3 --version
```

**การใช้:**
```env
# ในไฟล์ .env
DB_TYPE=sqlite
DB_PATH=./data/database.sqlite
```

### 2. MongoDB Local

**ข้อดี:**
- NoSQL - ยืดหยุ่นในโครงสร้าง
- เหมาะสำหรับข้อมูลที่ไม่มีโครงสร้างแน่นอน

**ข้อเสีย:**
- ต้องติดตั้งเซิร์ฟเวอร์ MongoDB
- ใช้หน่วยความจำมากกว่า SQLite

**ติดตั้ง:**

Windows:
```bash
# ดาวน์โหลดจาก https://www.mongodb.com/try/download/community
# เรียกใช้ Installer
# หรือใช้ Chocolatey
choco install mongodb-community

# เริ่มบริการ MongoDB
mongod
```

Mac:
```bash
brew tap mongodb/brew
brew install mongodb-community@5.0
brew services start mongodb-community@5.0
```

Linux (Ubuntu):
```bash
sudo apt-get install -y mongodb
sudo systemctl start mongodb
sudo systemctl enable mongodb
```

**การใช้:**
```env
# ในไฟล์ .env
DB_TYPE=mongodb
DB_CONNECTION_STRING=mongodb://localhost:27017/information_aggregator_local
```

### 3. PostgreSQL Local

**ข้อดี:**
- Relational Database ที่แข็งแกร่ง
- ดีที่สุดสำหรับข้อมูลโครงสร้างแน่นอน
- การทำงานเสถียรและเร็ว

**ข้อเสีย:**
- ต้องติดตั้งเซิร์ฟเวอร์
- ซับซ้อนกว่า SQLite

**ติดตั้ง:**

Windows:
```bash
# ดาวน์โหลดจาก https://www.postgresql.org/download/windows/
# เรียกใช้ Installer
# หรือใช้ Chocolatey
choco install postgresql

# ตั้งค่า service
# PostgreSQL จะทำงานเป็น Service อัตโนมัติ
```

Mac:
```bash
brew install postgresql@15
brew services start postgresql@15
```

Linux (Ubuntu):
```bash
sudo apt-get install postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

**การสร้างฐานข้อมูล:**
```bash
# เชื่อมต่อ PostgreSQL
psql -U postgres

# ในพรอมต์ psql
CREATE DATABASE information_aggregator_local;
\q
```

**การใช้:**
```env
# ในไฟล์ .env
DB_TYPE=postgresql
DB_HOST=localhost
DB_PORT=5432
DB_NAME=information_aggregator_local
DB_USER=postgres
DB_PASSWORD=your_password
```

### 📊 เปรียบเทียบ Local Databases

| ลักษณะ | SQLite | MongoDB | PostgreSQL |
|--------|--------|---------|------------|

| ความเร็ว | ✅ เร็ว | ✅ เร็ว | ✅✅ เร็วที่สุด |

| Concurrency | ❌ จำกัด | ✅ ดี | ✅✅ ดีที่สุด |
| โครงสร้าง | ✅ Fixed | ✅✅ Flexible | ✅ Fixed |
| เหมาะสำหรับ | เริ่มต้น | Rapid Dev | Production |
